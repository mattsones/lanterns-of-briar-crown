# Liam’s Game — Open Questions

This document tracks questions that should be answered later, but do not need to block the current repo setup.

## Story Questions

1. Is Lio Brindle alive, dead, or missing in a more complicated way?
2. When should the player first meet Mara?
3. Does Edden become an NPC later, or remain emotional weight for Hollis?
4. What exactly is Westroot?
5. Is the Briar Crown a faction, symbol, cult, route-network, or false authority movement?
6. How much should the player be able to shape the hero negatively through repeated choices?
7. Should Chapter 2 begin immediately after report-back, or after a rest/interlude?

## Worldbuilding Questions

1. What playable ancestry/race options exist?
2. Which ancestry has the strongest defensive identity for Rowan?
3. Are Mossbacks playable, rare NPC ancestry, or companion-only?
4. Are fox-like people part of the world at all, or should that accidental Rowan concept be discarded?
5. What is the old Lantern Road system historically?
6. Who maintained the true road marks?
7. Why did public maps stop showing Westroot?

## Gameplay Questions

1. How much combat should Chapter 1 include?
2. Should companion level-up happen automatically or through choices?
3. Should the battle pouch expand later?
4. Should the hero eventually choose a class/specialization?
5. Should skill checks use visible dice rolls every time?
6. Should failed skill checks affect hidden reputation/trust?
7. Should repeated selfish/cruel choices alter protagonist reputation?

## Art Questions

1. What are the exact hero customization options?
2. How will gear overlays work visually?
3. Will portraits be static images, layered assets, or generated variants?
4. How should the painted map align to the movement grid?
5. How much UI parchment texture is enough?
6. Should the Warden be softened visually for family-friendliness?
7. What style should item/skill icons use?

## Technical Questions

1. Should state remain in React state, or move to reducer/context?
2. When does the app need routing?
3. How should saves migrate between versions?
4. How should story flags be typed?
5. Should map interactions be data-driven?
6. Should dialogue become a data-driven graph?
7. Should Codex extract data first or simply scaffold the repo?

## Publishing Questions

1. Is the target a local-only prototype, itch.io-style web game, or private family game?
2. Will the game need mobile support?
3. Should it support controller/keyboard only?
4. Should save data export/import be added?
5. Should assets be optimized for web early or later?
